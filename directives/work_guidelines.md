# 제주 여행 도우미 작업 가이드라인 (Work Guidelines)

이 문서는 AI 어시스턴트가 프로젝트 작업을 수행할 때 반드시 준수해야 하는 핵심 원칙을 담고 있습니다. 모든 작업 시 이 내용을 먼저 읽고 참고하십시오.

## 1. 코드 수정 및 독립성 유지 (Code Isolation)
- 특정 기능을 수정할 때, 해당 기능과 직접적인 관련이 없는 다른 코드 블록은 수정하지 않습니다.
- 모든 작업은 기존의 다른 기능에 부정적인 영향이 가지 않도록 철저히 격리된 방식으로 수행되어야 합니다.
- 부득이하게 공통 모듈을 수정해야 하는 경우, 사전에 영향 범위를 분석하고 보고하십시오.

## 2. GitHub Push 제한 (Push Restriction)
- 작업 결과물을 GitHub 저장소에 Push하는 행위는 사용자의 명시적인 명령(예: "이제 깃허브에 푸시해줘")이 있을 때만 수행합니다.
- 로컬에서의 작업 완료 및 검증 이후에도 Push 명령이 내려지기 전까지는 원격 서버로의 전송을 대기하십시오.

## 3. 작업 전 체크리스트 (Pre-work Checklist)
- **API 응답 구조 확인**: JSON 형식인 경우 필드명 대소문자(전부 소문자 가능성) 및 데이터 타입(숫자/문자열)을 반드시 확인하십시오.
- **Worker 이중 인코딩**: API 호출 시 인증 오류가 나면 Worker에서 `%` 등 특수문자가 이중 인코딩되고 있지 않은지 확인하십시오.
- **허용 도메인**: 신규 API 호출 시 `worker.js`의 `ALLOWED_DOMAINS`에 등록되어 있는지 확인하십시오.
