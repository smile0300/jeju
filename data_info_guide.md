# 제주 여행 도우미 데이터 연동 가이드 (Data Info Guide)

이 프로젝트에서 각각의 실시간 정보를 어떻게 불러오는지에 대한 기술적 상세 가이드입니다.

## 1. 실시간 CCTV (📹)
- **방법**: 공공기관에서 제공하는 HLS 방식의 스트리밍 URL을 직접 연결하거나 YouTube 라이브 스트리밍을 임베드합니다.
- **기술**: 
  - `Hls.js`: 일반 브라우저에서 HLS(.m3u8) 영상을 재생하기 위해 사용합니다.
  - `CORS Proxy`: 보안 정책(CORS)으로 인해 직접 접근이 불가능한 스트리밍 소스는 자체 Cloudflare Worker 프록시(`PROXY_URL`)를 통해 우회하여 불러옵니다.
  - `YouTube IFrame API`: 유튜브 라이브 채널의 경우 IFrame을 통해 자동 재생 및 무음 설정을 적용합니다.

## 2. 날씨 정보 (🌤️)
- **방법**: **기상청 단기예보 및 중기예보 API**를 활용합니다.
- **출처**: [공공데이터포털 (data.go.kr)](https://www.data.go.kr)
- **상세**:
  - `getVilageFcst` (단기예보): 현재 기온, 강수 확률, 습도, 풍속 등을 3시간 단위로 가져옵니다.
  - `getWthrWrnMsg` (기상특보): 제주 지역(stnId: 184)에 발효된 풍랑, 강풍, 호우 등의 실시간 특보를 가져옵니다.
- **표시 변경** *(2026-03-18)*:
  - 날씨 카드 레이아웃을 **세로 배치**로 변경. 날씨 요약(`아이콘 + 기온 + 날씨설명`)이 상단, 세부 정보(습도·풍속·강수·강수확률)가 **2열 그리드**로 하단 표시.
  - 기상특보 없음 메시지 중문화: `"현재 발효된 기상특보가 없습니다"` → `"当前无气象特报"`

## 3. 한라산 등반 통제 (⛰️)
- **방법**: **제주특별자치도청 홈페이지** HTML 파싱(Scraping)으로 탐방로별 통제 상태를 추출합니다.
- **특이사항**: CORS 정책 우회를 위해 프록시 서버를 이용하며, 탐방로 이름과 상태를 매핑하여 `TRAIL_STATUS_MAP`으로 중문화합니다.

## 4. 항공편 운항 정보 (✈️)
- **방법**: **한국공항공사 실시간 운항 정보 API**를 연동합니다.
- **상세**:
  - `getArrFlightStatusList` (도착): 제주공항(CJU) 도착편 상태 조회.
  - `getDepFlightStatusList` (출발): 제주공항 출발편 상태 조회.
  - `REGION_AIRPORTS` Set으로 중화권 공항 코드만 필터링하여 표시.
- **표시 변경** *(2026-03-18)*:
  - **도시명 중문화**: `CITY_NAMES` 매핑 테이블 추가. 한국어 도시명(인천, 타이페이, 상하이 등) → 간체 자동 변환.
  - **항공사명 중문화**: `AIRLINE_NAMES` 매핑으로 항공사 코드 → 중문 명칭 자동 변환.
  - **운항 상태 번역**: `STATUS_MAP` 보완. `회항` → `返航/备降` 추가, 결항 배지에서 `✕` 기호 제거.
  - **최종 업데이트 시간**: 탭 상단 `#flight-update-time`에 데이터 로드 시각 `🕐 更新时间: YYYY.MM.DD HH:MM` 표시.

## 5. 축제 및 행사 (🎉)
- **제공처**: 비짓제주 (Visit Jeju)
- **Endpoint**: `https://api.visitjeju.net/vsjApi/contents/searchList`
- **Category**: `c5` (축제/행사)
- **표시 변경** *(2026-03-18)*:
  - **2026년 데이터 병합**: 일반 카테고리 검색 + 키워드 `"2026"` 검색 결과를 `Map(contentsid)`으로 중복 제거 후 병합.
  - **진행중 배지**: `_is2026` 플래그가 있는 항목에 `进行中` 배지 표시.
  - **필터링 기준**: 최근 12개월 이내 행사 OR 타이틀에 "2026" 포함 항목.
  - 각 카드는 비짓제주 상세 페이지(`contentsid`)로 직접 연결.

## 6. 습득물 정보 (🔍)
- **제공처**: 경찰청 (LOST112)
- **방식**: 공공데이터포털 OpenAPI (`LosfundInfoInqireService`, `LosPtfundInfoInqireService`)
- **수집 파라미터**:
    - `N_FD_LCT_CD=LCP000`: 제주특별자치도 통합 검색 코드
    - `START_YMD` / `END_YMD`: 사용자 선택 기간 (기본 어제)
    - `PRDT_CL_CD_01`: 물품 대분류 필터링
- **데이터 병합**: 경찰관서(`numOfRows=200`) + 포털기관(`numOfRows=200`) 병합, **최대 400건**.

## 7. 기능 요청 및 건의사항 저장 (Google Sheets)
- **방법**: **Google Apps Script (GAS)**를 백엔드로 활용하여 사용자 피드백을 수집합니다.
- **상세**:
  - 사용자로부터 입력받은 텍스트를 JSON 형태로 GAS 웹 앱 엔드포인트에 전송.
  - GAS가 수신된 데이터를 지정된 구글 시트의 마지막 행에 추가.
- **관리**: 상세 설정 방법은 `google_sheets_guide.md` 파일에 기록되어 있습니다.

---
**주요 설정 파일**: `script.js` 내의 `CONFIG` 객체에서 API 키 및 엔드포인트를 관리합니다.
